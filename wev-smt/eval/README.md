# Scalability evaluation (Day 9)

Plot-ready timing data for the SMT decision procedure across four parametric
weak-memory program families, nine sizes, and five models. Generated by
`wev.smt.cli.ScalabilitySweep` over the generators in `wev.smt.bench.ParametricPrograms`.

## Reproduce

```bash
mvn -q test                      # compile + copy Z3 natives (also runs the 16 unit tests)
mvn -q exec:exec@scalability-sweep
# constrained window (these committed CSVs were produced with):
mvn -q exec:exec@scalability-sweep -Dsweep.budget=8 -Dsweep.percall=30
```

`-Dsweep.budget` is the total wall-clock budget in **minutes** (spec default 30);
`-Dsweep.percall` is the per-call cap in **seconds** (spec default 60). The committed
CSVs use `budget=8, percall=30` because this run window is capped at ~10 min; the
consistency sweep (the headline) completes regardless, and separation is budget-limited
either way.

## Families (`ParametricPrograms`)

| family | shape | deps | events |
|--------|-------|------|--------|
| `LBChain` | n-thread load-buffering ring, each thread's write **really** depends on its read | data, `isSemantic=true` | 3n |
| `LBFakeChain` | identical structure, dependency is the identity (`r ^ r`) | data, `isSemantic=false` | 3n |
| `SBNThread` | n-thread store-buffering ring, each read misses its neighbour's write | — | 3n |
| `IRIWFan` | n independent writers, two readers reading all n locations in opposite orders | — | 4n |

`LBChain(2)`/`LBChain(3)` reproduce the hand-crafted corpus `LB`/`3.LB` structurally
(see `ParametricProgramsTest`). Sizes swept: `2, 3, 4, 5, 6, 8, 10, 12, 16`.

## `scalability-consistency.csv`

One row per (family, n, model). The **headline** metric is `fullexec_ms`: the
end-to-end time (formula build + solve) to decide whether the wired execution is
consistent under the model — the same check as the validation atlas.

| column | meaning |
|--------|---------|
| `family`, `n`, `events` | program identity and true event count (incl. initial writes) |
| `model` | SC / TSO / PSO / RA / WEAKEST |
| `verdict` | `ALLOWED` (SAT) / `FORBIDDEN` (UNSAT) / `TIMEOUT` |
| `fullexec_ms` | full-execution consistency decision time (the scaling metric) |
| `minwitness_size` | `findMinimalConsistent` cardinality; `-1` = timed out, `-2` = skipped past the budget deadline. (The minimum consistent sub-execution is trivially a singleton under the gated encoding, so this column is a sanity check, not the scaling story.) |
| `minwitness_ms` | time for the minimum-witness optimization (0 when skipped) |
| `used_mem_mb` | JVM used heap after the solve (`totalMemory - freeMemory`); coarse, GC-noisy |

## `scalability-separation.csv`

One row per (family, n, ordered model pair `allowedBy`/`forbiddenBy`), i.e. the full
`family × n × pair` grid (4 × 9 × 20 = 720 rows). Separation is the expensive CEGAR
search and is budget-limited.

| column | meaning |
|--------|---------|
| `status` | `SOLVED` (separating witness found), `NO_SEPARATION` (search exhausted, none exists in the encoding), `TIMEOUT` |
| `witnessSize` | active-event count of the minimum separating witness (blank unless `SOLVED`) |
| `ms` | wall-clock for the search. **`TIMEOUT` with `ms=0` = not started** (overall budget already spent); `ms>0` = the per-call search budget was hit |

## Findings (headline)

The decision procedure **scales polynomially, not exponentially**, in event count:

- **SC, TSO, WEAKEST: near-linear** — tens to low-hundreds of ms even at n=16
  (48–64 events). Notably **WEAKEST** (this prototype's contribution; the `jfCoherence`
  no-thin-air axiom encoded with the integer-layer trick, no reachability matrix) stays
  cheap as n grows.
- **PSO, RA: polynomial** (~1.8–2.0×/step median; non-uniform n-steps). These pay for
  the multi-segment `hb;eco?` / per-location coherence layers and are the cost drivers,
  reaching a few seconds at n=16 (IRIWFan/PSO the largest).
- **No model on any family shows sustained ≥2.5×/step growth beyond n=4** — the
  exponential stop-trigger does not fire.

Sanity (also enforced by `ParametricProgramsTest`): `LBChain(n)/WEAKEST = FORBIDDEN`
for all n ≥ 2 (real dependency cycle ⇒ thin-air, forbidden) while
`LBFakeChain(n)/WEAKEST = ALLOWED` for all n (fake dependency carries no semantic edge).
The two families share an identical event structure, so this is a pure dependency-content
effect at equal encoding cost.
